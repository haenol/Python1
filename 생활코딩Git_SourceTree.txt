3. 저장소 생성
저장소 = 프로젝트
특정 디렉토리를 저장소로 만드는 방법
->clone에서 원하는 디렉토리 설정하면 됨. 기존에 작업하던 디렉토리를 설정하려고 함
-근데 '유효한 소스 경로'가 아니라고 하면서 안 만들어짐-오류내용 구글링해봤는데 git command로 하는 것만 떠서 그걸로도 뭐 해봤는데 암튼 안됨-드림코딩님꺼 보면 답을 찾을 수 있지 않을까해서 보기 시작함-원래 터미널 안 깔고 기본 command창으로 하려고 했는데 보다보니 좋아보여서 깔았음-VScode도 연결하면 좋다길래 그거설정 또 한참걸림(드림코딩)-드림코딩님 내용 터미널창으로 따라가다 보니 숨겨진 git폴더를 원하는 위치에 만드는 방법 깨달아서(git init) 기존에 작업하던 디렉토리에 만듦(구글링한 걸로 해봤을 땐 안만들어짐)-소스트리에서 드디어 'Git 저장소입니다'뜸ㅜㅜㅜㅜ --------> 응 그냥 강의 제대로 안들어서 삽질~

3. 저장소 생성 = git 폴더 생성(git init)
 Create a repository 메뉴 이용
4. 버전 생성: 새 파일을 staging area로 드래그, 메모 작성 후 커밋
5. 하나의 버전, 여러 개의 파일: 커밋 한번에 보내기
6. 버전 쪼개기: 원하는 파일만 선택하여 staging area(커밋 대기상태)로 드래그하여 커밋
7. reset: 커밋을 수정하거나 돌아가고 싶을 경우(조심)
 - 방법: 돌아가고싶은 버전 오른쪽 클릭-reset~-hard 
 ---> 데이터가 해당 버전으로 돌아감(해당 버전 이후 데이터 모두 삭제)
8. revert: 기록을 훼손하지 않고 과거로 되돌림
 - 실수조차도 기록하는 것이 좋음
 - 현재 기록을 그대로 두면서 선택한 버전의 변경사항을 취소함
 - 따라서 역순으로 차례대로 취소해야한대.. 아직 다 이해 못함
 - 방법: 원하는 버전(revert7) 오른쪽 클릭-reverse commit-yes
 ---> 새로운 커밋 자동생성 Revert "revert7" => revert6 버전 
9. 시간여행: 파일들을 예전 특정 버전으로 되돌려놓고 싶다면
 - 방법: 원하는 버전 더블클릭-yes
 - 다시 현시점으로 돌아오고 싶을 때: branch(master) 더블클릭
(SHA) 커밋=버전의 정보를 보면 커밋id가 있음. 앞의 7자리정도가 SHA
10. 비교: 버전 사이의 차이점 비교
 - KDif3 프로그램: 두 커밋의 차이를 알려줌

수업을 마치며
 - diff tool을 이용하여 차이점을 보다 정교하게 비교--->의사결정에 도움
 - branch: 저장소를 여러 가지 상태로 공존할 수 있게
  ex) 고객사에게 프로그램 제공, 고객사마다 약간씩 다른 프로그램 필요
 - tag: 커밋id는 너무 복잡해서 쉽게 버전을 찾아갈 수 있는 도구
